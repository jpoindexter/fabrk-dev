# Plan: Remove Authentication System from Fabrk

## Overview
Remove the entire authentication/login system from the repository since the application doesn't use user accounts. This includes auth pages, API routes, middleware, database models, and all auth-related UI components.

## Scope
- Remove 35-40 authentication-related files
- Strip auth checks from ~50 API routes
- Remove auth middleware and route protection
- Delete 7 database models (User, Account, Session, VerificationToken, MFADevice, BackupCode, etc.)
- Remove auth-related navigation links
- Update environment variables and dependencies
- Clean up dashboard routes (remove or make public)

## Implementation Steps

### Phase 1: Remove Auth UI Components & Pages
**Files to delete:**
- `/src/app/(auth)/` - Entire directory with login, signup, register, forgot-password, reset-password, verify-email, magic-signin, accept-invite pages
- `/src/app/(dashboard)/` - Protected dashboard pages (or make public - TBD based on user preference)
- `/src/components/auth/` - Auth-related components (if any)

**Files to modify:**
- `/src/app/not-found.tsx` - Remove "Go to Dashboard", "Login", and "Sign Up" links
- `/src/app/error.tsx` - Remove any auth-related links (if present)
- `/src/components/landing/footer.tsx` - Remove "Login" link (line 64-65)

### Phase 2: Remove Auth API Routes
**Directories to delete:**
- `/src/app/api/auth/` - Entire auth API directory
- `/src/app/api/user/` - User management routes
- `/src/app/api/auth/mfa/` - MFA system routes

**Files to delete:**
- `/src/app/api/auth/[...nextauth]/route.ts` - NextAuth handler
- `/src/app/api/auth/forgot-password/route.ts`
- `/src/app/api/auth/register/route.ts`
- `/src/app/api/auth/reset-password/route.ts`
- `/src/app/api/auth/verify-email/route.ts`
- All other auth-related API routes

### Phase 3: Remove Auth Middleware & Protection
**Files to modify:**
- `/src/middleware.ts` - Remove route protection logic or delete file entirely
- All API routes with `auth()` checks - Remove authentication checks

**Pattern to find and remove:**
```typescript
const session = await auth();
if (!session?.user) {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
```

### Phase 4: Remove Database Models & Schema
**Files to modify:**
- `/prisma/schema.prisma` - Remove models:

  - Session
  - VerificationToken
  - MFADevice
  - BackupCode
  - Any other auth-related models

**After schema changes:**
```bash

```

### Phase 5: Remove Auth Configuration & Dependencies
**Files to delete:**
- `/src/lib/auth.ts` - NextAuth configuration

**Files to modify:**
- `/src/config.js` - Remove auth-related configuration
- `/src/lib/env.ts` - Remove auth environment variables
- `.env.example` - Remove auth-related variables

## Risks & Considerations
1. **Database Data Loss:** Removing User model will delete all user data (if database exists)
2. **Breaking Changes:** This is a major architectural change
3. **Dependent Features:** Many features may depend on user accounts
4. **Stripe Integration:** Payment tracking may need user reference
5. **Organizations:** Multi-tenancy won't work without users

## Rollback Strategy
- Git commit before starting changes
- Create a backup branch: `git checkout -b backup/before-auth-removal`
- Can revert with: `git reset --hard backup/before-auth-removal`
