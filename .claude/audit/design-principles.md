## WCAG 2.1 AA Summary

| Principle | Key Requirements |
|-----------|------------------|
| **Perceivable** | Alt text, captions, contrast |
| **Operable** | Keyboard, timing, navigation |
| **Understandable** | Readable, predictable, error help |
| **Robust** | Assistive tech compatible |

## Nielsen's 10 Heuristics (Summary)

1. Visibility of system status
2. Match real world language
3. User control & freedom
4. Consistency & standards
5. Error prevention
6. Recognition over recall
7. Flexibility & efficiency
8. Aesthetic & minimalist
9. Error recovery
10. Help & documentation

## Key Psychological Laws

| Law | Principle |
|-----|-----------|
| **Fitts** | Larger targets = faster selection |
| **Miller** | 7 +/- 2 items max in working memory |
| **Hick** | More choices = longer decisions |

## Quick Checklists

### UX Checklist
- [ ] Loading states visible
- [ ] Easy undo/cancel/exit
- [ ] Consistent patterns
- [ ] Error messages helpful

### Visual Hierarchy
- [ ] Important = largest/boldest
- [ ] Related items grouped
- [ ] Clear contrast
- [ ] Intentional whitespace

### Accessibility
- [ ] Color not sole indicator
- [ ] Contrast >= 4.5:1
- [ ] Touch targets >= 44px
- [ ] Keyboard accessible
