## Why a Setup Wizard?

Configuring a new project typically means:

- Reading documentation
- Finding example files
- Copying and editing configs
- Hoping you didn't miss anything

Fabrk's setup wizard handles it all interactively.

---

## Running the Wizard

```bash
npm run setup
```

The wizard guides you through:

1. Database configuration
2. Authentication setup
3. Payment provider selection
4. Email configuration
5. Theme selection

---

## Preview Mode

See what will change without modifying files:

```bash
npm run setup -- --dry-run
```

Output shows:
- Files that will be created
- Environment variables to set
- Configuration changes

## Configuration Object

The wizard generates a typed config:

```typescript
// src/config/index.ts
export const config = {
  app: {
    name: 'My SaaS',
    url: 'https://mysaas.com',
  },

  auth: {
    providers: ['github', 'google'],
  },

  payments: {
    provider: 'stripe',
    plans: ['free', 'pro', 'enterprise'],
  },

  email: {
    provider: 'resend',
    from: 'noreply@mysaas.com',
  },

  theme: {
    default: 'dracula',
    enableSwitcher: true,
  },
};
```

## Re-Running the Wizard

Need to change something? Run it again:

```bash
npm run setup
```

The wizard detects existing configuration and offers to:
- Keep current settings
- Update specific sections
- Reset everything

---

## Command-Line Options

```bash
# Preview without changes
npm run setup -- --dry-run

# Skip specific sections
npm run setup -- --skip-db --skip-email

# Non-interactive mode (use defaults)
npm run setup -- --defaults

# Reset all configuration
npm run setup -- --reset
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Wizard won't start | Run `npm install` first |
| Database won't connect | Check connection string format |
| OAuth fails | Verify callback URLs match |
| Email won't send | Check API key and domain verification |

---

## Best Practices

1. **Use dry-run first** - Preview before applying
2. **Keep secrets secure** - Never commit `.env.local`
3. **Test locally** - Verify each integration works
4. **Document changes** - Note custom configurations

## Getting Started

```bash
# Clone
git clone https://github.com/fabrk/fabrk-dev my-saas

# Install
cd my-saas && npm install

# Configure (interactive)
npm run setup

# Start building
npm run dev
```

Configuration, simplified.
