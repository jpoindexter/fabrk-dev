# UI/Styling Inconsistencies Audit Plan

## Objective
Conduct a comprehensive audit of all UI components in `src/components/ui/` to identify styling inconsistencies similar to the tabs component fix (border radius mismatches, tight padding, etc.).

## Context
Recent fix in tabs component:
- Border radius mismatch between outer container and active button
- Tight padding between button and container edge

## Audit Strategy

### Phase 1: Gather All UI Components
- List all `.tsx` files in `src/components/ui/` (excluding stories and test files)
- Categorize by type:
  - Container components (Card, Alert, Banner, etc.)
  - Interactive components (Button, Input, Checkbox, etc.)
  - Composite components (Accordion, Tabs, Dialog, etc.)
  - Data display (Table, List, Timeline, etc.)

### Phase 2: Search for Common Issues

#### Issue Type 1: Border Radius Mismatches
Search patterns:
- Components with nested interactive elements (buttons, badges inside containers)
- Look for combinations of:
  - `rounded-full` paired with `rounded-md` or `rounded-lg`
  - `rounded-none` on parent with `rounded-*` on children
  - Inconsistent radius values between container and content
- Wrappers with `overflow-hidden` that should match inner element radius

#### Issue Type 2: Tight Padding Issues
Search patterns:
- Very small padding values: `p-0`, `p-0.5`, `p-1` on container elements
- Wrapper/container components with minimal spacing around interactive elements
- Look for:
  - Gap/spacing issues between children
  - Cramped button/badge spacing inside containers
  - Input groups with tight padding
  - Icon + text spacing issues

#### Issue Type 3: Visual Alignment Issues
Search patterns:
- Height/width mismatches in flex containers
- Inconsistent gap values between siblings
- Vertical/horizontal centering issues

### Phase 4: Search Patterns to Execute

#### Grep searches for border radius issues:
1. `className="[^"]*rounded-full[^"]*rounded-[a-z]` - conflicting radius
2. `className="[^"]*rounded-none[^"]*rounded-[a-z]` - conflicting radius
3. `overflow-hidden.*rounded-` - verify radius matching
4. `\[border-radius:[^]]*\].*rounded-` - mixed CSS variable and tailwind radius

#### Grep searches for padding issues:
1. `className="[^"]*\bp-0\b[^"]*` - on containers/wrappers
2. `className="[^"]*\bp-1\b[^"]*` - on containers/wrappers
3. `className="[^"]*\bp-0\.5\b[^"]*` - on containers/wrappers
4. Looking for small `px-`, `py-` with `gap-` combinations
5. Button/badge inside container with minimal spacing

#### Visual inspection patterns:
1. Components with nested interactive elements
2. Wrappers applying `overflow-hidden`
3. Flex/grid containers with asymmetric padding
4. Icon + text components spacing
5. State variants (hover, active, disabled) padding consistency

### Phase 5: Documentation & Reporting

#### Report Format for Each Issue
```
**File:** `src/components/ui/[filename].tsx`
**Line:** [line number]
**Component/Section:** [Component name or specific part]
**Issue Type:** [Border Radius Mismatch | Tight Padding | Alignment Issue | Other]
**Current Code:** [Snippet showing the problem]
**Problem:** [Why this is inconsistent/problematic]
**Suggested Fix:** [Specific recommendation with exact classNames]
**Severity:** [Critical/High/Medium/Low]
**Visual Impact:** [How this affects the UI]
```

#### Severity Levels
- **Critical:** Breaks visual alignment, appears visibly broken, padding crushes content
- **High:** Noticeable visual inconsistency with other components, creates cramped look
- **Medium:** Subtle inconsistency that affects polish, minor spacing differences
- **Low:** Very minor differences, minimal visual impact

### Phase 6: Execution Steps

1. **Search for border radius patterns** across all UI component files
2. **Search for padding patterns** especially on container/wrapper elements
3. **Read specific files** where issues are identified for context
4. **Cross-reference** similar components to identify patterns
5. **Group findings** by issue type and severity
6. **Document** with file path, line number, and actionable fix

3. **Read** - Deep inspection of specific components identified

## Notes for Audit
- Use `tabs.tsx` as reference for recently fixed issues
- Look for patterns (some issues may repeat across similar components)
- Consider both nested elements and sibling spacing
- Pay attention to responsive behavior (different padding at breakpoints)
- Check state variants (hover, active, disabled, focus)
- Verify icon + text spacing is consistent
- Look at button sizes and how padding scales with size variants
