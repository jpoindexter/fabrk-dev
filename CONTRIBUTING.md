# Contributing to Fabrk

Thank you for your interest in contributing to Fabrk!

## For Customers

If you purchased Fabrk, you have full rights to modify the code for your projects. This guide is for contributing back to the main Fabrk boilerplate.

## Bug Reports

Please open an issue on GitHub with:

- Clear description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Environment details (Node version, OS, browser)

## Feature Requests

Open an issue describing:

- The feature and its use case
- How it would benefit other Fabrk users
- Any implementation ideas you have

## Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run the test suite (`npm test`)
5. Run the linter (`npm run lint`)
6. Commit with conventional commit messages (`feat:`, `fix:`, `docs:`, etc.)
7. Push to your fork
8. Open a Pull Request

## Code Style

- Follow the existing code style
- Use TypeScript strict mode
- Follow the terminal design system (see `docs/08-design/DESIGN_SYSTEM.md`)
- Use design tokens instead of hardcoded colors
- Add tests for new features

## Questions?

Open an issue or contact support@fabrk.dev
